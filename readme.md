# Part 1: Programming Exercise: Simple RAG from Scratch

In this repository you will find a lightweight Retrieval-Augmented Generation (=RAG) system, that was implemented using python. By creating vector embeddings for each line, and using, in memory cosine similarity search, to filter for relevant context, and using a local LLM, this RAG can generate answers via Ollama that are free of hallucination.

## Setup

Please ensure you're using a version of Python thats 3.8+.

Check via 
```
python --version
```

You also need the required packages:
```bash
pip install ollama
```
Also pull:
```
ollama pull hf.co/CompendiumLabs/bge-base-en-v1.5-gguf
ollama pull hf.co/bartowski/Llama-3.2-1B-Instruct-GGUF
```

### How to run my programm:

To run the implemented RAG, please execute `demo.py`:
```
python demo.py
```
Firstly, you will have to answer what database you want to be loaded, please either choose `cat-facts.txt` or `cat-db-fiction-confusing.txt`, where `cat-facts.txt` will be choosen by default. 

Secondly, you need to select the amount for N (=3 in default case), where N is the Top-N chunks that the system retrieves. 

Thirdly, ask a question. Then, the pipeline will output into the terminal: A response generated by the LLM based purely on the retrieved chunks with their cosine similarity.

Lastly, you may either enter 'exit' or 'quit' to close the session.
